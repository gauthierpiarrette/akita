After a search, rows the server saw fetched are highlighted — winners and decoys, indistinguishable. The query, scores, and ranking never appear here.
1 · Embed on-device. The sentence-transformer runs on the key holder’s side; plaintext never leaves.
2 · Seal. Text is AES-GCM encrypted; embeddings are CKKS-encrypted (128-bit, hardened params).
3 · Blind search. The server multiplies ciphertexts it cannot read — one op per segment, no rotations.
4 · Decrypt at the edge. Only the key holder decrypts scores and picks the winners.
Honest v0 notes: search download scales with corpus size (~230 KB/segment); fetch patterns are padded with decoys pending a PIR stage; key loss means memory loss — that is what “the operator cannot read it” costs.
