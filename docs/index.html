Akita gives AI apps per-user memory and search the operator provably cannot read — text sealed with AES-GCM, embeddings encrypted with CKKS, keys that never leave the user's side. Recall is fast enough to use, and every number below was measured, not promised.

How it works

1 · Embed on-device The sentence-transformer runs on the key holder's side; plaintext never leaves.
2 · Seal Text is AES-GCM encrypted; embeddings are CKKS-encrypted (128-bit, hardened parameters).
3 · Blind search The server multiplies ciphertexts it cannot read — one operation per segment, no rotations.
4 · Decrypt at the edge Only the key holder decrypts scores and picks the winners.
Stated plainly, because trust products should say it themselves: Akita protects memory storage and search — once your app decrypts a memory into an LLM prompt, privacy depends on where inference runs. Search download scales with corpus size in v0; fetch patterns are padded with decoys pending a PIR stage; key loss means memory loss. Full caveats in the security notes.
